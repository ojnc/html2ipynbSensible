<h1><a name="introduction"></a>Introduction / Use case</h1>
<p><a href="https://saphanacloudservices.com/hana-cloud/"  target="_blank" >SAP HANA Cloud</a> is the latest evolution of SAP HANA. Released in March 2020, it combines the technology known from the on-premise version of SAP HANA with cloud benefits such as scalability.</p>
<p>It contains many of the known SAP HANA engines, such as the <a href="https://help.sap.com/viewer/product/apl/latest/en-US"  target="_blank" >Automated Predictive Library (APL)</a>. The APL, which is a highly automated ML framework, has also been evolving and now includes a &#8220;Gradient Boosting Classification&#8221;. This new Gradient Boosting Classification automates the process of finding an ML model, similar to the previous APL. The new Gradient Classification approach however tends to result in stronger models. Since the APL product group is focussing further improving this approach, we suggest working with the new &#8220;Gradient Boosting Classification&#8221;, as we do in this blog.</p>
<p>The APL can be called through SQL or directly from Python. In this blog we make use of Python and the corresponding library <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/latest/en-US/hana_ml.html"  target="_blank" >hana_ml</a>, which simplifies the calling of SAP HANA functionality. We will call the APL, but the library can also call the <a href="https://help.sap.com/viewer/2cfbc5cf2bc14f028cfbe2a2bba60a50/latest/en-US"  target="_blank" >Predictive Algorithm Library (PAL)</a>.</p>
<p>As example to work with the Gradient Boosting Classification, we will predict the affinity of a Banking customer to sign up for a term deposit, in which money can be locked for a certain period and accrues higher interest rates.</p>
<p>&nbsp;</p>
<h1><a name="setup_hana"></a>Setup SAP HANA Cloud</h1>
<p>To implement this Bank Marketing scenario you need access to SAP HANA Cloud. This SAP HANA Cloud with the following configuration:</p>
<ul>
<li>It must have at least 3 virtual CPUs (vCPUs). The current SAP HANA Cloud Trial comes only with 2 vCPUs. In that trial it is not possible to increase the number of vCPUs. Therefore you must create your own instance of SAP HANA Cloud in the SAP Cloud Platform Cockpit as described in this <a href="https://www.youtube.com/playlist?list=PL48QrZZyMuAYJ5TxjTMK2_y5_5-RKJC2P"  target="_blank" >playlist</a> from the SAP HANA Academy, which also contains plenty of additional material.</li>
<li>Your SAP HANA Cloud must have the Script Server enabled to be able to use the Automated Predictive Library (APL). For a new SAP HANA Cloud instance the Script Server can be <a href="https://help.sap.com/viewer/db19c7071e5f4101837e23f06e576495/2020_03_QRC/en-US/f7febb16072b41f7ac90abf5ea1d4b86.html"  target="_blank" >enabled</a> when creating the instance. For existing sytems, the Script Server can be activated with a <a href="https://help.sap.com/viewer/db19c7071e5f4101837e23f06e576495/cloud/en-US/918e714867e1409da47fa01ce03ba8f3.html"  target="_blank" >Service Request</a>.</li>
<li>Your SAP HANA Cloud must permit connections from your IP address, e.g. by selecting &#8220;Allow all IP addresses&#8221;.</li>
</ul>
<p>The &#8220;<a href="https://saphanacloudservices.com/hana-cloud/learning-article/what-is-sap-hana-cloud"  target="_blank" >Getting Started with SAP HANA Cloud</a>&#8221; guide also gives an excellent overview, just bear in mind that you cannot use the trial environment to implement this blog&#8217;s hands-on scenario. <a href="https://people.sap.com/denys.kempen"  target="_blank" >Denys van Kempen</a> has also put together a huge <a href="https://blogs.sap.com/2020/03/28/getting-started-with-sap-hana-cloud/" >collection of SAP HANA Cloud material</a>. In case you might be a little puzzled about the different version of SAP HANA that have been around, Denys has also created a <a href="https://blogs.sap.com/2020/05/09/sap-hana-cheat-sheet/" >SAP (HANA) Cheat Sheet</a>.</p>
<p>There is also a Learning Article on <a href="https://saphanajourney.com/hana-cloud/learning-article/using-jupyter-notebooks-with-sap-hana-cloud/"  target="_blank" >Using Jupyter Notebooks with SAP HANA Cloud</a> that helps with the first steps.</p>
<p>To continue hands-on, you need to know:</p>
<ul>
<li>The SAP HANA Cloud endpoint (e.g. xyz.hanacloud.ondemand.com)</li>
<li>The HTTPS port, which is always 443</li>
<li>The password for the DBADMIN user</li>
</ul>
<h1><a name="setup_python"></a>Setup local Python</h1>
<p>You need an environment to execute Python code. In case you already have an environment to work with, please use it. In case you are new to Python, we suggest installing <a href="https://www.anaconda.com/products/individual#Downloads"  target="_blank" >Anaconda</a>, which will install and configure everything that is needed.</p>
<p>Once you have downloaded and installed Anaconda, open the Anaconda Navigator and select JupyterLab.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2020/06/10-Anaconda-Navigator.png" alt="" data-index="2" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>An alternative, and much faster, approach to open JupyterLab is through the Anaconda Prompt. Open the Anaconda Prompt, then type: jupyter lab</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2020/06/20-Anaconda-Prompt.png" alt="" data-index="3" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>If you haven&#8217;t worked with JupyterLab so far, not to worry It&#8217;s easy to find your way around. One very useful introduction for the first steps is this <a href="http://www.rpgroup.caltech.edu/ncbs_pboc/code/t0b_jupyter_notebooks.html"  target="_blank" >tutorial from the California Institute of Technology (Caltech</a>).</p>
<p>With Python at your fingertips, create a new Python 3 Notebook and install SAP&#8217;s <a href="https://pypi.org/project/hana-ml/"  target="_blank" >hana_ml</a> library, which provides the connectivity and push-down to the SAP HANA Cloud.</p>

<h1><a name="create_user"></a>Create SAP HANA user</h1>
<p>Test the connectivity by running a very simple SELECT statement. You will need the SAP HANA Cloud endpoint, HDB port and the DBADMIN user&#8217;s password. Here we see the password in clear text only for a quick test. Later on, we will securely store the passwords.</p>
<pre class="language-python"><code>import hana_ml.dataframe as dataframe

# Instantiate connection object

df_category_profit.style.format({'Profit':'{0:,.2%}'}).hide(axis='index')</code></pre>
<p>&nbsp;</p>
<p>And bring these details into a plot.</p>
<pre class="language-python"><code>import seaborn as sns
import matplotlib.pyplot as plt
from matplotlib.ticker import PercentFormatter
plt.figure()
bplot = sns.barplot(data = df_category_profit, x = 'Category', y = 'Profit', color = '#1f77b4')
bplot.set_title(variable_name)
bplot.set_xticklabels(bplot.get_xticklabels(), rotation = 45);
bplot.yaxis.set_major_formatter(PercentFormatter(1))</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2020/07/BarChartProfits-1.png" alt="" data-index="21" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>Categories on the left hand side in the above chart have the highest proportion of the target population. This means the purchasing affinity (influence) was highest when the customer was contacted in March, September, October or December.</p>
<p>The categories are sorted in descending order of the proportion / influence. When the influence (y-axis) becomes negative, the target proportion of the category is lower than the overall&#8217;s population. This means people were less likely to purchase. People were not keen to buy the product in May. In summary: the further left the category on the x-axis, the higher the affinity.</p>
<p>Plot the category significance for some other variables. AGE is interesting for example. The relationship between age and purchasing affinity is not linear.</p>
<p>For those who want to look a little deeper into the model, view the correlations between the input variables. The APL can calculate a full correlation matrix, as all input variables are encoded numerically. APL can deal with highly correlation variables, so you don&#8217;t have to go this deep. However, in case there are highly correlated variables you can consider dropping these from the input variables to speed up the calculation time.</p>
<pre class="language-python"><code>df_correlations = gbapl_model.get_indicators().filter("KEY = 'CorrelatedVariable'").collect()
df_correlations = df_correlations[['VARIABLE', 'DETAIL', 'VALUE']]
df_correlations['VALUE'] = df_correlations['VALUE'].astype(float).round(3)
df_correlations.columns = ['1st Variable', '2nd Variable', 'Coefficient']
df_correlations = df_correlations.sort_values(by = ['Coefficient'], ascending = False)
df_correlations.style.hide(axis='index')</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" loading="lazy" src="https://blogs.sap.com/wp-content/uploads/2020/06/193-correlations.png" alt="" width="268" height="113" data-index="22" data-id="plbox-1113330" /></p>
<p>There is nothing that would concern us here, but these three variables have fairly strong correlations to each other.</p>
<p>Another optional deep-dive is a look at the interactions, which the model found between the variables.</p>
<pre class="language-python"><code>df_interactions = gbapl_model.get_indicators().filter("KEY = 'InteractionValue' and to_char(VALUE) &lt;&gt; '0'").collect()
df_interactions = df_interactions[['VARIABLE', 'DETAIL', 'VALUE']]
df_interactions['VALUE'] = df_interactions['VALUE'].astype(float).round(3)
df_interactions.columns = ['1st Variable', '2nd Variable', 'Interaction']
df_interactions = df_interactions.sort_values(by = ['Interaction'], ascending = False)
df_interactions.style.format({'Interaction':'{0:,.3f}'}).hide(axis='index')</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" loading="lazy" src="https://blogs.sap.com/wp-content/uploads/2020/06/196-interactions-1.png" alt="" width="240" height="292" data-index="23" data-id="plbox-1113330" /></p>
<p>No strong interactions were found. We are just showing them here for you to know, that the APL&#8217;s new GradientBoostingBinaryClassifier captures these automatically. These interactions can also be very relevant in interpreting the model&#8217;s logic.</p>
<p>Let&#8217;s have a look at one further detail, that we can use later on. The model will be using SHAP scores for the predictions. With the complete trained model still available at the moment, we can find out the threshold score, which will be used to separate positive and negative predictions.</p>
<pre class="language-abap"><code>df_threshold = gbapl_model.get_indicators().filter("KEY = 'Threshold'").collect()
df_threshold = df_threshold[['VARIABLE', 'VALUE']]
df_threshold['VALUE'] = df_threshold['VALUE'].astype(float)
df_threshold.columns = ['Target', 'Score Treshold']
df_threshold.style.hide(axis='index')</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" loading="lazy" src="https://blogs.sap.com/wp-content/uploads/2020/06/198-shap-threshold.png" alt="" width="159" height="47" data-index="24" data-id="plbox-1113330" /></p>
<p>Scores below -1.15998 will lead to negative predictions. Scores above will lead to positive predictions.</p>
<p>&nbsp;</p>
<p>This score can be converted to the corresponding probability value:</p>
<pre class="language-python"><code>import math
score = df_threshold.iloc[0, 1]
1 / (1 + math.exp(-score))</code></pre>
<p>&nbsp;</p>
<p>We are now familiar with the model and could use it right away for predictions. However, it&#8217;s also a good moment to save the model first to the SAP HANA Cloud.</p>
<pre class="language-python"><code>from hana_ml.model_storage import ModelStorage
MODEL_SCHEMA = 'ML' # HANA schema in which models are to be saved
model_storage = ModelStorage(connection_context=conn, schema = MODEL_SCHEMA)
gbapl_model.name = 'Bank Marketing Model'
model_storage.save_model(model=gbapl_model, if_exists = 'replace')</code></pre>
<p>&nbsp;</p>
<h1><a name="apply"></a>Apply ML model</h1>
<p>Let&#8217;s use the model to create predictions! Create a new Python 3 Notebook called “40 Apply ML model”.</p>
<p>Connect to SAP HANA Cloud.</p>
<pre class="language-python"><code>import hana_ml.dataframe as dataframe
conn = dataframe.ConnectionContext(userkey = 'MYHANACLOUD',
                                   encrypt = 'true')</code></pre>
<p>&nbsp;</p>
<p>Create the SAP HANA DataFrame, which points to the table with the entities that need to be predicted. Sort the data for reproducibility.</p>
<pre class="language-abap"><code>df_remote_new = conn.table(table = 'BANKMARKETING_TOPREDICT', schema = 'ML').sort("CUSTOMER_ID", desc = False)
df_remote_new.head(5).collect()</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2020/06/200-to-predict-1.png" alt="" data-index="25" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>Load the trained ML model.</p>
<pre class="language-python"><code>from hana_ml.model_storage import ModelStorage
MODEL_SCHEMA = 'ML' # HANA schema from where models are to be loaded
model_storage = ModelStorage(connection_context = conn, schema = MODEL_SCHEMA)
gbapl_model = model_storage.load_model(name = 'Bank Marketing Model', version = 1)</code></pre>
<p>&nbsp;</p>
<p>Apply the model to predict the behaviour of the two customers. For each person we obtain an overall probability of purchasing the product (proba_PURCHASE) and the yes / no prediction.</p>
<pre class="language-abap"><code>gbapl_model.set_params(extra_applyout_settings =
                        {'APL/ApplyExtraMode': 'Advanced Apply Settings',
                         'APL/ApplyDecision': 'true',
                         'APL/ApplyProbability': 'true',
                         'APL/ApplyPredictedValue': 'false'
                        })
df_remote_predict = gbapl_model.predict(df_remote_new)
df_predict = df_remote_predict.head(10).collect()
df_predict.columns = [hdr.replace('gb_', '') for hdr in df_predict.columns] # shorten column names
df_predict.style.format({'proba_PURCHASE':'{0:,.3f}'}).hide(axis='index')</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" loading="lazy" src="https://blogs.sap.com/wp-content/uploads/2020/07/205-predicted-5.png" alt="" width="363" height="101" data-index="26" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>If you would like to understand which characteristics of a customer contributed towards a positive or negative prediction, you can leverage the so called SHAP values.</p>
<pre class="language-python"><code>gbapl_model.set_params(extra_applyout_settings={'APL/ApplyExtraMode': 'Individual Contributions'})
df_remote_predict = gbapl_model.predict(df_remote_new)
df_predict = df_remote_predict.head(10).collect()
df_predict.columns = [hdr.replace("gb_", "") for hdr in df_predict.columns] # Shorten column names
df_predict</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2020/06/210-predict-1-1.png" alt="" data-index="27" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>Each prediction relates to a SHAP value (score_PURCHASE), which itself is a sum of the SHAP values of the individual predictors, which are also shown. The higher the overall score, the higher the probability the person will purchase the product. Earlier on we saw the threshold that is used to differentiate between positive and negative predictions (1.15998 in this case).</p>
<p>&nbsp;</p>
<p><span style="color: #ff0000"><span style="color: #000000">If you want to better understand how the variables contributed to these individual predictions (local explainable ML / AI, just rearrange the data format and visualise them in a plot.</span></span></p>
<pre class="language-python"><code>df_contributions_single = df_predict[df_predict['CUSTOMER_ID'] == 2]
df_contributions_single = df_contributions_single.melt(id_vars = ['CUSTOMER_ID', 'PREDICTED', 'score_PURCHASE'])
df_contributions_single.columns = ['CUSTOMER_ID', 'PREDICTED', 'SCORE', 'VARIABLE', 'CONTRIBUTION']
df_contributions_single = df_contributions_single.sort_values(by = ['CONTRIBUTION'], ascending = [False])
df_contributions_single.style.format({'CONTRIBUTION':'{0:,.3f}'}) 

import seaborn as sns
import matplotlib.pyplot as plt
import numpy as np

%matplotlib inline 
plt.figure()
bar_color = np.where(df_contributions_single['CONTRIBUTION'] &gt; 0 ,'green', 'red')
bplot = sns.barplot(data = df_contributions_single, x = 'VARIABLE', y = 'CONTRIBUTION',  palette = bar_color.tolist())
bplot.set_title('Contributions')
bplot.set_xticklabels(bplot.get_xticklabels(), rotation = 90)
plt.axhline(y = 0, color = 'grey', linewidth = 1)
plt.ylim(-2.5, 2.5);</code></pre>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" decoding="async" loading="lazy" src="https://blogs.sap.com/wp-content/uploads/2020/06/220-shap-plot.png" alt="" width="417" height="402" data-index="28" data-id="plbox-1113330" /></p>
<p>&nbsp;</p>
<p>The above plot shows the variable&#8217;s contribution for customer number 2. Towards the left the variables that contributed most strongly to a positive prediction are shown. The &#8220;contrib_constant_bias&#8221; shown in the plot is not a variable. It is an element of the trained model, which can be interpreted as an average, which is adjusted by the individual variables.</p>
<p>Write the predictions to a table in SAP HANA Cloud, where other processes or Analytical tools like SAP Analytics Cloud can pick them up.</p>
<pre class="language-python"><code>df_remote_predict.save(where=('ML', 'BANKMARKETING_PREDICTED'), 
                       table_type = 'COLUMN', 
                       force = True)</code></pre>
<p>&nbsp;</p>
<p>Just close the connection, and you are done.</p>
<pre class="language-python"><code>conn.close()</code></pre>

<p><a href="https://www.sap.com/products/data-intelligence.html"  target="_blank" >SAP Data Intelligence</a> provides the platform to deploy HANA ML models into production. Its &#8220;ML Scenario Manager&#8221; for example combines Jupyter Notebooks for creation with graphical pipelines for deployment.</p>

<p>It is important that your HANA ML operators use a recent version of the Python library <a href="https://pypi.org/project/hana-ml/"  target="_blank" >hana_ml</a> (version 2.5.x). This is already the case with SAP Data Intelligence Cloud.</p>

                <p>Hello David, There are a few options to avoid hardcoding the password.<br />
You could store the password safely in the hdbuserstore. The section "Create SAP HANA user" describes an example. The connection is then established with</p>
<pre class="language-python"><code>conn = dataframe.ConnectionContext(userkey = 'MYHANACLOUD', encrypt = 'true')</code></pre>
<p>Alternatively, If the password is not specified in the code, hana_ml will prompt for it.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="alignleft ds-lightbox__img" src="https://blogs.sap.com/wp-content/uploads/2020/07/PasswordPrompt.png" alt="" width="390" height="155" data-index="1" data-id="clbox-545738" /></p>
<p>You could also store the password elsewhere, ie in a config file, and retrieve / pass it through code.</p>

Feel free to put me on CC, happy to help along if I can.</p>

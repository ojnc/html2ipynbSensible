    <h1 class="ds-heading ds-heading--m ds-blog-post__title">Hands-On Tutorial: Leverage SAP HANA Machine Learning in the Cloud through the Predictive Analysis Library</h1>
    <div class="ds-blog-post__social-stats-wrapper">
      <div class="ds-social-stats ">
        <span class="ds-social-stats__stat">
          <i class="ds-icon ds-icon--comments"></i>7        </span>
        <span id="post-likes-counter" class="ds-social-stats__stat">
          <i class="ds-icon ds-icon--likes-outline"></i>32        </span>
        <span class="ds-social-stats__stat">
          <i class="ds-icon ds-icon--views"></i>3,258        </span>
      </div>
          </div>
    <div class="ds-blog-post__body ds-editor-content">
      <p>The hard truth is that many machine learning projects fail to get set into production. It takes time and real effort to move from a machine learning model to a real business application. This is due to many different reasons, for example:</p>
<ol>
<li>Limited data access</li>
<li>Poor data quality</li>
<li>Small computing power</li>
<li>No version control</li>
</ol>
<p>Of course, we can’t save the world with just one Hands-On tutorial, but we can at least try to make the life of a data scientist a little easier. <strong>In this blog post we will tackle these challenges by bringing the opensource world and SAP world together.</strong> In a nutshell, there will be no movement of training data from SAP HANA Cloud to our Python environment. In the following, we will refer to our SAP HANA Cloud simply as HANA. Of course, you can access the data and sample code under the following <a href="https://github.com/SAP-samples/hana-cloud-ml-challenges"  target="_blank" >GitHub repository</a>. But let us start from the beginning!</p>
<ol>
<li><strong>How can we directly access the data in our HANA? </strong></li>
</ol>
<p>As data scientists we feel very comfortable in our RStudio or Python Jupyter Notebooks. We want to stay in our used environment and don’t want to move around in too many different tools. Or even worse, we must copy the code or data from A to B, working on our local laptop with different csv files or other formats. Hence, the great news is that we can directly connect through the R &amp; Python clients with our HANA. Let us look at an example.</p>
<p>In our Python script we first install and import the following library:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/1-163.png" alt="" data-index="1" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/2-59.png" alt="" data-index="2" data-id="plbox-1284380" /></p>
<p>The <a href="https://pypi.org/project/hana-ml/"  target="_blank" >hana_ml library</a> enables us to directly connect to our HANA. To leverage it’s full potential we have to make sure that our user has the following policies assigned:</p>
<ol>
<li>AFL__SYS_AFL_AFLPAL_EXECUTE_WITH_GRANT_OPTION</li>
<li>AFL__SYS_AFL_APL_AREA_EXECUTE</li>
<li>AFLPM_CREATOR_ERASER_EXECUTE</li>
</ol>
<p>Set your HANA host, port, user, password and encrypt to true:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/3-54.png" alt="" data-index="3" data-id="plbox-1284380" /></p>
<p>Execute the following command to connect to your HANA:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/4-45.png" alt="" data-index="4" data-id="plbox-1284380" /></p>
<p>We can hide our login credentials through the <a href="https://help.sap.com/viewer/b3ee5778bc2e4a089d3299b82ec762a7/2.0.05/en-US/dd95ac9dbb571014a7d7f0234d762fdb.html"  target="_blank" >Secure User store</a> from the HANA client and don’t have them visible in clear text. In our command prompt we execute the following script:</p>
<p>C:\Program Files\SAP\hdbclient&gt;hdbuserstore -i SET MYHANACLOUD “YOURENDPOINT:PORT” YOURUSERNAME</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/5-neu.png" alt="" data-index="5" data-id="plbox-1284380" /></p>
<p>Then back in our Python script we can use the HANA key to connect:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/6-41.png" alt="" data-index="6" data-id="plbox-1284380" /></p>
<p>Now, let us upload a local dataset and push it directly into HANA. Make sure you change the path to your local directory.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/7-37.png" alt="" data-index="7" data-id="plbox-1284380" /></p>
<p>Before we bring our local dataset into HANA, we must execute some transformations. We change the columns to upper string and add a unique Product ID to the data. This ID will later be used as a key in our machine learning algorithms, which are directly running in our HANA.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/8-34.png" alt="" data-index="8" data-id="plbox-1284380" /></p>
<p>Next, let us create a HANA dataframe and point it to the table with the uploaded data.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/9-31.png" alt="" data-index="9" data-id="plbox-1284380" /></p>
<p>Great job! We tackled our first challenge in our machine learning use case. Since, we are now able to directly connect to our HANA from our used environment, let us move to the next task.</p>
<ol start="2">
<li><strong>How can we explore our data and react to data quality issues early? </strong></li>
</ol>
<p>Data understanding and preparation take up a lot off time during a machine learning use case. Increasing the data quality can drive every data scientist crazy, due to so many reasons. Hence, let us look at some options we have now through the hana_ml library. Right away our <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.04/en-US/hana_ml.dataframe.html"  target="_blank" >HANA dataframe</a> provides us with different functionalities. Therefore, let us control the data size and if all the variable types are set correctly:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/10-36.png" alt="" data-index="10" data-id="plbox-1284380" /></p>
<p>Our data upload was complete but since the variable Quality only contains 0s and 1s, it was falsely set to type integer. The variable is binary and labels all products of bad quality with a 1 and 0 otherwise. Since this is by definition a categorical variable, we transform it to type NVARCHAR with the following command.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/11-33.png" alt="" data-index="11" data-id="plbox-1284380" /></p>
<p>To better understand the data, we can derive a description of different statistical attributes or even get a whole <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.05/en-US/hana_ml.visualizers.html#dataset-report-label"  target="_blank" >data report</a> through the following commands:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/12-34.png" alt="" data-index="12" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/13-35.png" alt="" data-index="13" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/14-27.png" alt="" data-index="14" data-id="plbox-1284380" /></p>
<p>If we now find any errors, we have many different functions on hand, staying in our Python environment. For example, we can execute sql commands directly in our Python script or use many more functions available with our <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.04/en-US/hana_ml.dataframe.html"  target="_blank" >HANA dataframe</a>.</p>
<ol start="3">
<li><strong>How can we leverage the computing power of our HANA in our machine learning use case? </strong></li>
</ol>
<p>At some point we must bring the data and the algorithms together to train our machine learning models. Since our HANA is not only an in-memory database, we don’t have to collect the data into our local environment. Instead we can leverage the native machine learning libraries directly in our HANA. Therefore, let us split the data into a training and testing set through the <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.05/en-US/hana_ml.algorithms.pal.html"  target="_blank" >Predictive Analysis Library</a> (PAL).</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/15-18.png" alt="" data-index="15" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/16-20.png" alt="" data-index="16" data-id="plbox-1284380" /></p>
<p>Further, let us train and optimize a <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.04/en-US/hana_ml.algorithms.pal.html#hana_ml.algorithms.pal.trees.RandomForestClassifier"  target="_blank" >random forest algorithm</a> to classify the quality of a product. First, we set the numbers of trees very high, to see where the Out of Bag error converges. After optimizing the numbers of trees, we will take a closer look at the variables considered at each split.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/17-18.png" alt="" data-index="17" data-id="plbox-1284380" /></p>
<p>Now, we can apply our trained model on the testing set.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/18-18.png" alt="" data-index="18" data-id="plbox-1284380" /></p>
<p>For each observation we can see the predicted class and of course the confidence of our random forest model. In addition, we can compute the confusion matrix or a have a look at the variable importance.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/19-19.png" alt="" data-index="19" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/20-17.png" alt="" data-index="20" data-id="plbox-1284380" /></p>
<p>The idea is to keep the heavy lifting in our HANA and only collect small sets to for example evaluate our random forest model. Therefore, we can collect the OOB error and use opensource libraries to visualize the results:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/21-12.png" alt="" data-index="21" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/22-12.png" alt="" data-index="22" data-id="plbox-1284380" /></p>
<p>Our trained model converges after around 700 trees. Hence, let us set the parameter to 801 trees to leave a little buffer. Further, let us optimize the number of variables considered at each split by iterating through each possible value.</p>
<p><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/23-12.png" alt="" data-index="23" data-id="plbox-1284380" /></p>
<p style="overflow: hidden;margin-bottom: 0px">Let’s plot the result:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/24-14.png" alt="" data-index="24" data-id="plbox-1284380" /></p>
<p>After 2 variables considered at each split the error increases and we must be careful not to overfit the data. Therefore, let us train our optimal random forest model.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/25-11.png" alt="" data-index="25" data-id="plbox-1284380" /></p>
<p>Congratulation, you created your first machine learning model directly in HANA. Now, let’s move to our last pain point.</p>
<ol start="4">
<li><strong>How can we save and create different versions of our results? </strong></li>
</ol>
<p>After the evaluation of our machine learning model a last very crucial step is still missing. We must deploy our model into the IT landscape and make our new insights accessible to the business, such that they can incorporate this information into their decision making. Our huge advantage is now that our machine learning model already resides in HANA. We can create a <a href="https://help.sap.com/doc/1d0ebfe5e8dd44d09606814d83308d4b/2.0.04/en-US/hana_ml.model_storage.html"  target="_blank" >Model Storage</a> where we can save our trained model and load it again for consumption. Hence, please execute the following script:</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/26-9.png" alt="" data-index="26" data-id="plbox-1284380" /></p>
<p>Let’s view the saved model in our model storage.</p>
<p style="overflow: hidden;margin-bottom: 0px"><img class="aligncenter ds-lightbox__img" decoding="async" src="https://blogs.sap.com/wp-content/uploads/2021/02/27-11.png" alt="" data-index="27" data-id="plbox-1284380" /></p>
<p>Of course, we can now save additional tables created based on our model and consume them in different frontend tools like the SAP Analytics Cloud. For example, have a look at the following blogs:</p>
<ul>
<li><a href="https://www.sapanalytics.cloud/guided_playlists/design-formatting-and-aesthetics/"  target="_blank" >Story design, formatting, and aesthetics</a></li>
<li><a href="https://www.sapanalytics.cloud/learning/"  target="_blank" >SAP Analytics Cloud Learning</a></li>
<li><a href="https://saphanajourney.com/sap-analytics-cloud/learning-article/hands-on-tutorial-sap-analytics-cloud-r-visualization/"  target="_blank" >Hands-On Tutorial SAP Analytics Cloud, R Visualization</a></li>
</ul>
<p>In addition, if you like to try the <a href="https://help.sap.com/viewer/3c53bc7b58934a9795b6dd8c7e28cf05/2020_04_QRC/en-US/2e7660d216144e78bb809ce0f9c6382a.html"  target="_blank" >Automated Predictive Library (APL)</a> or are a big R fan, try out the following Hands-On tutorials:</p>
<ul>
<li><a href="https://blogs.sap.com/2020/07/27/hands-on-tutorial-automated-predictive-apl-in-sap-hana-cloud/" >Hands-On Tutorial: Automated Predictive (APL) in SAP HANA Cloud</a></li>
<li><a href="https://blogs.sap.com/2020/05/29/hands-on-tutorial-leverage-sap-hana-embedded-machine-learning-through-an-r-shiny-app/" >Hands-On Tutorial: Leverage SAP HANA embedded Machine Learning through an R Shiny App</a></li>
<li><a href="https://blogs.sap.com/2020/06/10/hands-on-tutorial-becoming-the-chief-data-cook-with-rstudio-and-sap-hana/" >Hands-On Tutorial: Becoming the Chief Data Cook with RStudio and SAP HANA</a></li>
